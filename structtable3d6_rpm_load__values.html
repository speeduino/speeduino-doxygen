table3d_value_t & table3d6RpmLoad_values::value_at ( table3d_dim_t  linear_index)
inline

Direct access to table value element from a linear index.

Since table values aren't laid out linearly, converting a linear offset to the equivalent memory address requires a modulus operation.

This is slow, since AVR hardware has no divider. We can gain performance in 2 ways:
1. Forcing uint8_t calculations. These are much faster than 16-bit calculations
2. Compiling this per table type. This encodes the axis length as a constant thus allowing the optimising compiler more opportunity. E.g. for axis lengths that are a power of 2, the modulus can be optimised to add/multiply/shift - much cheaper than calling a software division routine such as __udivmodqi4

THIS IS WORTH 20% to 30% speed up

This limits us to 16x16 tables. If we need bigger and move to 16-bit operations, consider using libdivide.
